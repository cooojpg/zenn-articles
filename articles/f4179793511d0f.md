## はじめに

AWSの公式ドキュメントにはCloudWatch Agentで取得できるメトリクスの一覧が掲載されていますが、**記載されているのはLinuxのメトリクスのみ**です。

https://docs.aws.amazon.com/ja_jp/AmazonCloudWatch/latest/monitoring/metrics-collected-by-CloudWatch-agent.html

私はそれに気づかずにドキュメント作成を進めていました。調べてみてもWindowsのメトリクスを取得する記事があまりなかったので、どんなものが取れるのか整理してみました。

これからWindowsの監視をするよーって方の参考になれば幸いです。

# CloudWatch Agent — Windows メトリクス一覧

> **注意事項**
> - Windows メトリクスはすべて `Unit: None`（Windowsパフォーマンスカウンターのイベントをそのまま解析するため、計算・変換は行われない）
> - メトリクス名はオブジェクト名とカウンター名をスペースでつないだもの（例: `Processor` オブジェクトの `% Idle Time` → CloudWatch上では `Processor % Idle Time`）
> - コンソール表示名と実際のメトリクス名が異なる場合あり（例: `LogicalDisk % Free Space` の実際のメトリクス名は `% Free Space`）
> - デフォルト namespace: `CWAgent`
> - インスタンス指定で特定ドライブ・NIC・CPUコアを絞り込み可能（例: `LogicalDisk` のインスタンスに `C:` を指定）

収集レベル凡例: 🟢 Basic / 🟡 Standard / 🔴 Advanced

収集レベルは、CloudWatch Agentの定義済みメトリクスセット（predefined sets）に対応しています。

| レベル | 説明 |
|---|---|
| 🟢 Basic | 最小限の監視 |
| 🟡 Standard | Basicに加えて、CPU・ディスクI/Oなどの詳細メトリクスが追加 |
| 🔴 Advanced | Standardに加えて、TCP接続数やさらに細かいI/Oメトリクスも取得 |

つまり、表中の収集レベルは「このメトリクスを取得するには最低どのレベルの設定が必要か」を示しています。

## Windowsのディスク監視における注意点 — Linuxとの違い

### 物理ディスク容量は取得できない

Linuxでは `disk_used_percent` などで物理ディスク全体の使用率を簡単に取得できますが、**Windowsではデフォルトで物理ディスクの容量メトリクスは取得できません**。

取得できるのは `LogicalDisk` オブジェクト経由の**論理ディスク**のみです。EC2インスタンス（Windows Server）の場合、一般的に以下の2つの論理ディスクが認識されます。

| 論理ディスク | 用途 |
|---|---|
| `C:` | メインのシステム・データ用ドライブ |
| `HarddiskVolume2` | 起動専用の裏方ディスク（EFI System Partition等）の可能性が高い |

### HarddiskVolume2 とは

`HarddiskVolume2` はドライブレターが割り当てられていないボリュームで、**Windows の起動プロセスに使われる内部パーティション**である可能性が高いと考えられます。ただし、ドキュメントにも明確な記載はなく、確かな情報は見つかりませんでした。

### PhysicalDisk オブジェクトについて

`PhysicalDisk` オブジェクトのメトリクスも存在しますが、取得できるのは **I/O パフォーマンス系のメトリクスのみ**です（前述の表を参照）。

- `% Disk Time` — ディスクがI/O要求を処理していた時間の割合
- `Disk Write Bytes/sec` / `Disk Read Bytes/sec` — 書き込み・読み取りバイト数
- `Disk Writes/sec` / `Disk Reads/sec` — 書き込み・読み取り操作数

**ディスクの空き容量・使用率といったキャパシティ系のメトリクスは `PhysicalDisk` には含まれていません。**

### 推奨するディスク監視の方針

| 対象 | 方針 | 理由 |
|---|---|---|
| `C:` ドライブ | **主要な監視対象**としてアラート設定 | システム・アプリケーションデータが配置されるメインドライブ |
| `HarddiskVolume2` | **補助的にメトリクスだけ取得**（アラートは不要） | 起動用パーティションの可能性が高く、通常は容量変動しない |

設定例:

```json
{
  "metrics": {
    "metrics_collected": {
      "LogicalDisk": {
        "measurement": ["% Free Space"],
        "metrics_collection_interval": 60,
        "resources": ["*"]
      }
    }
  }
}
```

> `"*"` を指定するとすべての論理ディスク（`C:` や `HarddiskVolume2` など）が取得対象になります。アラートは `C:` ドライブのみに設定し、`HarddiskVolume2` はダッシュボードでの可視化にとどめるのが実用的です。

## おわりに

AWSでのWindows監視に関する情報は、Linuxと比べるとまだまだ少ないのが現状です。特にディスク監視については「物理ディスクの容量が取れない」「論理ディスクに見慣れない `HarddiskVolume2` がある」など、**Linuxの感覚で設定すると想定と異なる結果になる**ポイントがあります。

この記事を通して、早い段階で「WindowsのCloudWatch監視はLinuxとは違う」ということに気づいていただければ幸いです。
