## はじめに

はじめまして。25卒でクラウドエンジニアとして社会人生活をスタートしました。

入社から約1年が経ち、自分なりに走り続けてきたこの1年間を振り返りつつ、これからZennで発信していくことについて書いていきます。

## 自己紹介

- **本業**：メーカー系子会社のクラウドエンジニア
- **副業**：生成AI系スタートアップ企業のインフラエンジニア

本業ではメーカーグループのクラウド基盤に関わりながら、副業では生成AIプロダクトを支えるインフラに携わっています。

## 1年間で取得した資格

この1年で以下の資格を取得しました。

| 資格 | 分類 |
| --- | --- |
| 基本情報技術者試験 | IPA |
| 応用情報技術者試験 | IPA |
| AWS Certified Cloud Practitioner (CLF) | AWS |
| AWS Certified Solutions Architect - Associate (SAA) | AWS |
| AWS Certified Developer - Associate (DVA) | AWS |
| AWS Certified Cloud Operations - Associate (COA) | AWS |

IPAの基本情報・応用情報でIT全般の基礎を固めつつ、AWSのアソシエイト資格を3つ取得してクラウドの知識を広げてきました。

## 1年を振り返って

正直、あっという間の1年でした。

本業では実際のクラウド環境に触れながら、設計・構築・運用を一通り経験することができました。資格の勉強で得た知識と実務が結びつく瞬間が何度もあり、インプットとアウトプットの両輪を回すことの大切さを実感しています。

副業では生成AI系のスタートアップという、本業とはまた違ったスピード感の環境に身を置くことで、技術力だけでなく自走力や判断力も鍛えられました。

## これからのこと

2年目に入り、これからはアウトプットにも力を入れていきたいと思っています。

このZennでは主に以下のテーマで発信していく予定です。

- **AWS関連の技術記事** — 資格勉強や実務で得た知識の整理・共有
- **AWSの勉強方法** — 各資格をどう勉強して取得したかの具体的な方法
- **副業のはじめ方** — 新卒エンジニアがどうやって副業を始められたのか、その過程やノウハウ
- **得た知見** — インフラや生成AI周りの話

「資格は取ったけど、実際どう使うの？」「エンジニアの副業ってどうやって始めるの？」といった部分を、自分の経験をもとに具体的に書いていければと思います。

少しずつにはなりますが、同じように駆け出しのエンジニアの方や、AWSに興味がある方の参考になれば嬉しいです。

## おわりに

まだまだ未熟ですが、インプットとアウトプットを続けながら成長していきます。
